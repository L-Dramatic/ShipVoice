## 总体原则

1. 先保证完整链路每天都可运行，再逐步替换成真实模型。
2. 每个加分点必须落到可展示证据：实验表、截图、视频、消融对比或代码模块。
3. 不把微调当成孤立亮点，微调必须服务于“领域回答质量”和“安全拒答质量”。
4. 任何外部依赖都保留 fallback，避免答辩现场因为 GPU、网络或模型服务失败而崩掉。

## 需要用户或组员协助的事项

这些事项我会在需要时停下来找你确认：

1. GPU/服务器：微调 Qwen、跑 SenseVoice/CosyVoice 时需要你提供可用机器或确认本机环境。
2. API key：如果使用远程 LLM 或云模型，需要你提供环境变量方式，不要直接发明文 key。
3. 真实音频：ASR 评测需要组员录制 20-50 条测试音频。
4. 组员名单和分工：最终报告和 PPT 必须写真实分工。
5. 学号姓名：最终压缩包命名和邮件主题需要组长信息。
