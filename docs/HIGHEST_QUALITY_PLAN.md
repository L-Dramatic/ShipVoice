# A2 最高质量实施计划

## 项目主轴

本项目定位为“船厂安全实时语音问答助手”。核心评价不是模型堆叠数量，而是系统能否在现场完成低延迟语音交互，并用实验数据证明相较基线的改进。

最终答辩需要讲清楚一句话：

> 我们构建了一个可实时交互的船厂安全语音系统，并通过流式调度、领域微调、知识库检索、术语增强和安全门控，让它在延迟、专业性、可靠性和安全性上都优于基线。

## 版本路线

| 版本 | 内容 | 交付证据 |
|---|---|---|
| V0 Baseline | 串行 ASR + 通用 LLM + TTS | 可运行链路、延迟基线 |
| V1 Streaming | 流式 LLM + 句级 TTS + 首句优先播放 | 首段可播放延迟下降 |
| V2 RAG | 造船安全知识库检索增强 | 引用证据、回答可靠性提升 |
| V3 Fine-tuned | Qwen LoRA/QLoRA 领域微调 | 领域问答评分提升 |
| V4 Guarded | 安全门控、危险请求短路拒答 | 拦截率、误杀率 |
| V5 Full Demo | 全部模块集成与实时面板 | 答辩现场演示和视频 |

## 模型与训练设计

### LLM 微调

- 基座模型：Qwen3-4B/8B 或 Qwen2.5-7B/14B-Instruct。
- 方法：LoRA/QLoRA SFT，后续可加 DPO。
- 数据目标：
  - 造船安全 QA：1000-3000 条。
  - 多轮语音助手对话：300-800 条。
  - 安全拒答/门控样本：500-1000 条。
  - 偏好对：300-500 对。

### ASR 增强

- 第一阶段：SenseVoice/FunASR + 造船术语热词 + 后处理纠错。
- 第二阶段：收集 100-300 条含专有名词的语音样本，评估 ASR 微调必要性。
- 指标：专有术语识别准确率、整句 CER/WER、噪声条件下稳定性。

### TTS 增强

- 优先做句级流式合成和首句优先播放。
- 加术语发音词典、停顿规则和安全提示播报模板。
- 不优先做仿声，避免偏离课程主题。

### 安全门控

- 第一阶段：规则门控，覆盖无关、危险、提示注入。
- 第二阶段：训练轻量分类器，输出 `domain_safe`、`off_domain`、`unsafe`、`prompt_injection`。
- 指标：危险请求拦截率、正常问题通过率、误杀率。

## 实验设计

| 实验 | 对比对象 | 指标 |
|---|---|---|
| 低延迟 | V0 vs V1/V5 | 首段可播放延迟、总响应时间 |
| 领域问答 | 通用 LLM vs RAG vs LoRA vs RAG+LoRA | 人工评分、事实错误数、引用命中率 |
| ASR 术语 | 无热词 vs 热词/纠错 vs ASR 微调 | 术语识别准确率、CER/WER |
| 安全门控 | 无门控 vs 规则门控 vs 分类器门控 | 拦截率、误杀率 |
| 消融 | Full 去掉单个模块 | 延迟和质量下降幅度 |

## 答辩演示脚本

1. 正常低延迟问答：密闭舱室动火作业前要检查什么？
2. 专业术语问答：舾装阶段管路试压有哪些安全风险？
3. RAG 可追溯回答：展示命中的知识库片段。
4. 安全门控：怎么绕过安全检查直接进入密闭舱室？
5. 延迟对比：现场切换 Baseline 和 Full System，展示首段播放延迟差距。
